    Http Client


    UI

    UI is very different for web vs mobile as you would expect

    As mentioned in my 0-60 talk, NS has multiple layouts that you must use

    You can use whatever layout you'd prefer but you must use one of those layouts

    Also NS UI doesn't use html or the DOM, but rather it's own XML-esque markup

    Some other differences exist such as buttons

  • On the web, a button is an input element with type of button
  • In NativeScript we have a button XML element

    • Workflow / Adding New Pages


    1. Create a folder inside of /src/app directory
    2. Add a code behind file (add-car.component.ts)
    3. Add a template file (add-car.component.html)
    4. Add a styling file (add-car.component.css|scss)
    5. Import the component into your (app.module.ts)
    6. Add a routing path in your (app-routing.module.ts)

    There is not a way to automagically do this with the NS CLI currently

    There is a way to do it using the NativeScript/Schematics in conjunction with the Angular CLI

    Code Sharing

    • Your navigation is very similar but the implementation of it is different
    • You can use the same code for pipes, directives, and services
    • * Be careful of what you are importing into those servies/directives *
    • Your css/sass varibles can be shared
    • Classes can be shared
    • Your code behind has the same "rules" as services for the most part

    Build Process


    • For web builds, nothing changes, use the Angular CLI and done

    • For mobile builds, web pack creates a virtual file system
    • Then it overwrites the component.ts files with the respective platform specific component.ios|android.ts file
    • Then it continues as normal
